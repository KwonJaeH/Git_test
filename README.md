## About CVIDG
  ### Background
  * VP and GP communication S/W protocols use protocols defined by themselves.  
    This communication protocol-based functional message definition (DataMap) is stored in Codebeamer or Excel.  
    There are between 2000 and 2500 messages defined for each project.
  
  ### Problem  
  * We may experience inefficiency for redundant data storage.  
    Human errors may occur because people write one by one when coding the data map.  

  ### Solution 
  * Development efficiency will be enhanced through automation of data map code generation based on communication protocol between VP and GP.  
    - Remove duplicate jobs.  
    - Preventing Human Errors.  
    - Compilation and runtime error prevention through validation of data.  
  * Generate code for reviewed data map items only to ensure consistency.
  * Provides GUI for ease of use and usability.
